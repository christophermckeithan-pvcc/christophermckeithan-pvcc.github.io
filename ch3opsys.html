Operating Systems Technology
TERM DEFINITION
operating system The software that controls the overall operation of the computer including file management, the user interface, and the program execution environment
job queue A queue is a storage organization in which objects (in this case, jobs) are ordered in first-in, first-out (abbreviated FIFO and pronounced "FI-foe) fashion.
FIFO Objects in the queue that are removed from the queue order in which they arrived.
interactive processing Operating systems that allow a program being executed to carry on a dialogue with the user through remote terminals.
time-sharing Operating systems that provide service to multiple users at a time
multiprogramming A technique in which time is divided into intervals and then the execution of each job is restricted to only one interval at a time
application software The programs for performing tasks that are common to computer systems in general.
system software The programs for performing tasks particular to the machine's utilization. IE spreadsheets, database systems, desktop publishing systems, program development software, and games.
utility software A collection of software units that consists of programs performing activities that are fundamental to computer installations but not included in the operating system. Consists of software units that extend the capabilities of an operating system.
graphical user interface (GUI) Modern systems that perform tasks in which objects can be manipulated, such as files and programs, and are represented pictorally through the display of icons. These systems allow users to issue commands by using one of several common devices like mouse and keyboard.
shell Older user interfaces that communicated with users through textual messages using a keyboard and a monitor screen
kernel The internal part of an operating system that contains software components that perform the very basic functions required by the computer installation.
device driver Collections of components of the kernal that are software units that communicate with controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine. Each is uniquely designed for its particular device, such as a printer, and translates generic requests into the more technical steps required by the device assigned to the driver.
