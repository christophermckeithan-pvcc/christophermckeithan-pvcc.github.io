All About GitHub

  • Q. What is a version control system?
    A. A system that tracks changes made to files over time, allowing users to revert back to previous version of a project, see who made specific edits, and collaborate effectively on a project by keeping a history of all changes made to the code.


  • Q. What is Git?
    A. a free, open-source version control system that helps developers track changes to their code and collaborate on projects


  • Q. What is GitHub?
    A. A web-based platform that allows developers to store, share, and collaborate on code


  • Q. What is the difference between Git and GitHub?
    A. Git is a version control system that allows developers to track changes in their code. GitHub is a web-based hosting service for git repositories. In simple terms, you can use git without Github, but you cannot use GitHub without Git.


  • Q. Who started GitHub and how was it started?
    A. The idea for GitHub originated with Linus Torvalds, the creator of the Linux kernel operating system, in 2005. Torvalds created Git, an open-source distributed version control system (DVCS), as a way to maintain Linux. Git allows users to view a program's source code, development history, bug tracking, and integration plans.


  • Q. What company owns it now?
    A. In 2018, Microsoft acquired GitHub for $7.5 billion, making it a subsidiary of Microsoft Corporation.


  • Q. How much does a GitHub account cost?
    A. GitHub offers free and paid plans for storing and collaborating on code. Some plans are available only to personal accounts, while other plans are available only to organization and enterprise accounts.


  • Q. What is the Octocat?
    A. The Octocat is the mascot for GitHub, the world's largest software development platform. It's a mythical creature that's part octopus and part cat, and is often depicted with eight arms and cat-like features.


Git/Github terms
TERM Definition
repository A fundamental element of GitHub that stores files, code, and revision history for each file. It's a centralized location that allows multiple developers to work together on a project
commit Similar to saving a file that's been edited, a commit records changes to one or more files in your branch. Git assigns each commit a unique ID, called a SHA or hash, that identifies: The specific changes. When the changes were made. Who created the changes.
fork a new repository that shares code and visibility settings with the original “upstream” repository. Forks are often used to iterate on ideas or changes before they are proposed back to the upstream repository, such as in open source projects or when a user does not have write access to the upstream repository.
push The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repo.
pull requests In a pull request, collaborators can review and discuss the proposed set of changes before they integrate the changes into the main codebase. Pull requests display the differences, or diffs, between the content in the source branch and the content in the target branch.
workflows Automated sequences that you create within your GitHub repository to handle tasks like building, testing, and deploying your software.
issues A built-in issue-tracking system in GitHub repositories that allow users to report bugs, request new features, note performance problems, get feedback, collaborate on code projects, plan tasks, and track and manage a project.
raw button With the raw view, you can view or copy the raw content of a file without any styling.
blame button The git blame command is used to examine the contents of a file line by line and see when each line was last modified and who the author of the modifications was.
